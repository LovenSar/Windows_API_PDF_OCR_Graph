DIAGNOSIS_STATUS枚举描述提交到帮助程序类的假设的结果，其中已确定组件的运行

PROBLEM_TYPE 枚举 (ndhelper.h) 项目2024/02/29
PROBLEM_TYPE 枚举描述帮助程序类指示存在的问题类型。
语法 C++
typedef enum tagPROBLEM_TYPE {   PT_INVALID = 0,
  PT_LOW_HEALTH = 1,   PT_LOWER_HEALTH = 2,
  PT_DOWN_STREAM_HEALTH = 4,   PT_HIGH_UTILIZATION = 8,   PT_HIGHER_UTILIZATION = 16,
  PT_UP_STREAM_UTILIZATION = 32} PROBLEM_TYPE;
常量  
PT_INVALID 值： 0
PT_LOW_HEALTH 值：1
组件本身存在低运行状况问题。 在此组件所依赖的本地组件中未发现任何问题。
PT_LOWER_HEALTH 值： 2
此组件所依赖的本地组件中存在低运行状况问题。
PT_DOWN_STREAM_HEALTH 值： 4
低运行状况问题在于此组件依赖的现成组件。
PT_HIGH_UTILIZATION 值： 8
组件的资源被高度利用。 在此组件所依赖的本地组件中未发现高利用率。

组件高利用率问题的原因来自依赖它的本地组件。
PT_UP_STREAM_UTILIZATION 值： 32
组件高利用率问题的原因上游依赖于它的网络组件。要求

REPAIR_STATUS 枚举 (ndhelper.h) 项目2024/03/13
REPAIR_STATUS枚举描述帮助程序类尝试修复选项的结果。
语法 C++
typedef enum tagREPAIR_STATUS {   RS_NOT_IMPLEMENTED = 0,
  RS_REPAIRED,   RS_UNREPAIRED,   RS_DEFERRED,   RS_USER_ACTION} REPAIR_STATUS;
常量  
RS_NOT_IMPLEMENTED 值： 0
帮助程序类未实现修复选项。 RS_REPAIRED
帮助程序类修复了问题。 RS_UNREPAIRED
帮助程序类尝试修复问题，但验证表明修复操作未成功。 RS_DEFERRED
此时，帮助程序类无法执行修复。 RS_USER_ACTION
帮助程序类需要用户执行操作，然后才能继续修复。要求
